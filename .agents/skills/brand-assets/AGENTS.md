# Brand Assets Skill Governance

- Read `SKILL.md`, then only the routed reference needed for the asset being changed.
- Keep `references/*.md` as one-line bullets with observable rules and rationale; do not record one-off task history.
- For brand asset changes, run `bun run typecheck`; also run `bun run build:share` when public share viewer metadata or static social assets are touched.
- Visually inspect changed logo/favicon/OGP assets before shipping; type checks do not prove raster output quality.
