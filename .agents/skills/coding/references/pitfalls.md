# Coding Pitfalls

- `@pierre/diffs` `File` can collapse under dev React StrictMode with a cold grammar because its first instance leaves an unfinished shadow-DOM `<pre>`, while the remounted instance treats any `<pre>` as hydrated and skips the render completion callback; keep the `FILE_OPTIONS.onPostRender` unmount cleanup in `apps/syokan/src/catalogs/Code/index.tsx`, removing only a `<pre>` without `[data-code]`, and re-test a cold-grammar first mount in a real dev browser after pierre upgrades because `<pre>` and `[data-code]` are internal DOM details.
- In `apps/syokan/server/fileSource.ts`, watch the file and re-arm after `rename` on macOS because Bun directory watches miss descendant content changes; on Linux watch the parent without basename filtering because file watches miss inode swaps and coalesced atomic-save events may report only the temporary filename.
- Keep `apps/syokan/bunfig.toml` as a symlink to `../../bunfig.toml`: Bun reads `bunfig.toml` only from the current working directory, so the package-scoped dev process otherwise misses `bun-plugin-tailwind` and serves CSS without utilities.
- If an edited frontend module does not appear in the Bun HTML-import dev server even though source and typecheck are current, restart the dev server; touching the file or reloading the browser may retain a stale module bundle, so source-versus-DOM mismatch is the useful signal.
